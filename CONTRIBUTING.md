# Contributing Guidelines

## Commit Message Format
type: short description in english

## Commit Types
feat:      adding a new feature
fix:       fixing a bug
docs:      changes in documentation only
style:     code formatting, no logic change
refactor:  code change with no new feature or bug fix

## Commit Examples
feat: add user registration endpoint
feat: create product listing page
fix: correct password hashing error
fix: resolve cart total calculation bug
docs: update README with setup steps
style: format login component
refactor: simplify order controller logic

## Branch Naming
Always create your branch from dev, not main.

feature/person2-auth
feature/person3-database
feature/person4-frontend

## Pull Request Rules
- Always work on your own feature branch
- When done, open a Pull Request to dev branch
- Never push directly to main or dev
- Add a short description in your Pull Request explaining what you did
